## VS Code 插件数据存储到 globalStorage

### VS Code 提供的存储 API

插件开发时，通过 **`ExtensionContext`** 对象访问存储，VS Code 自动帮你管理路径。

---

### 1. `globalState` - 键值对存储（最简单）

```typescript
// 在 activate 函数中获取 context
export function activate(context: vscode.ExtensionContext) {
  // ✅ 写入数据
  await context.globalState.update("myKey", { name: "test", count: 42 })

  // ✅ 读取数据
  const data = context.globalState.get<{ name: string; count: number }>("myKey")
  console.log(data?.name) // 'test'

  // ✅ 删除数据
  await context.globalState.update("myKey", undefined)

  // ✅ 设置哪些 key 可以同步到云端（Settings Sync）
  context.globalState.setKeysForSync(["myKey"])
}
```

> 底层自动存到 `globalStorage/你的插件ID/state.vscdb`（SQLite 数据库）

---

### 2. `globalStorageUri` - 存文件（大数据）

```typescript
import * as vscode from "vscode"
import * as fs from "fs"
import * as path from "path"

export async function activate(context: vscode.ExtensionContext) {
  // 获取 globalStorage 路径
  const storageUri = context.globalStorageUri
  console.log(storageUri.fsPath)
  // 输出: /Library/Application Support/Code/User/globalStorage/你的发布者.插件名/

  // ✅ 确保目录存在
  await vscode.workspace.fs.createDirectory(storageUri)

  // ✅ 写入文件
  const fileUri = vscode.Uri.joinPath(storageUri, "mydata.json")
  const content = Buffer.from(JSON.stringify({ hello: "world" }), "utf8")
  await vscode.workspace.fs.writeFile(fileUri, content)

  // ✅ 读取文件
  const raw = await vscode.workspace.fs.readFile(fileUri)
  const data = JSON.parse(Buffer.from(raw).toString("utf8"))
  console.log(data) // { hello: 'world' }

  // ✅ 删除文件
  await vscode.workspace.fs.delete(fileUri)
}
```

### 3. 封装一个简单的存储工具类

```typescript
import * as vscode from "vscode"

export class Storage {
  private storageUri: vscode.Uri

  constructor(private context: vscode.ExtensionContext) {
    this.storageUri = context.globalStorageUri
  }

  // 初始化（确保目录存在）
  async init() {
    await vscode.workspace.fs.createDirectory(this.storageUri)
  }

  // 写入 JSON 文件
  async write<T>(filename: string, data: T): Promise<void> {
    const fileUri = vscode.Uri.joinPath(this.storageUri, filename)
    const content = Buffer.from(JSON.stringify(data, null, 2), "utf8")
    await vscode.workspace.fs.writeFile(fileUri, content)
  }

  // 读取 JSON 文件
  async read<T>(filename: string): Promise<T | null> {
    try {
      const fileUri = vscode.Uri.joinPath(this.storageUri, filename)
      const raw = await vscode.workspace.fs.readFile(fileUri)
      return JSON.parse(Buffer.from(raw).toString("utf8")) as T
    } catch {
      return null // 文件不存在时返回 null
    }
  }

  // 删除文件
  async delete(filename: string): Promise<void> {
    const fileUri = vscode.Uri.joinPath(this.storageUri, filename)
    await vscode.workspace.fs.delete(fileUri)
  }
}

// 使用
export async function activate(context: vscode.ExtensionContext) {
  const storage = new Storage(context)
  await storage.init()

  await storage.write("config.json", { theme: "dark", fontSize: 14 })
  const config = await storage.read<{ theme: string }>("config.json")
  console.log(config?.theme) // 'dark'
}
```

### 存储方式对比

| 方式                 | API                        | 适合场景              | 大小限制   |
| -------------------- | -------------------------- | --------------------- | ---------- |
| **globalState**      | `context.globalState`      | 少量键值配置          | 建议 < 1MB |
| **globalStorageUri** | `context.globalStorageUri` | 大文件、缓存、数据库  | 无限制     |
| **secrets**          | `context.secrets`          | Token、密码等敏感数据 | 小数据     |

---

### secrets 存储敏感数据

```typescript
// 专门存储敏感信息（加密存储）
await context.secrets.store("apiToken", "sk-xxxxx")
const token = await context.secrets.get("apiToken")
await context.secrets.delete("apiToken")

// 监听变化
context.secrets.onDidChange((e) => {
  console.log("secret changed:", e.key)
})
```

### 目录结构示意

```
globalStorage/
└── 你的发布者.插件名/
    ├── state.vscdb        # globalState 自动生成
    ├── config.json        # 你手动写入的文件
    ├── cache/             # 你创建的子目录
    │   └── data.bin
    └── mydata.json
```

> 💡 **推荐原则**：
>
> - 小配置 → 用 `globalState`
> - 大数据/文件 → 用 `globalStorageUri`
> - 密钥/Token → 用 `secrets`
